# Profile Plan

## Task Mode

`create`

## Primary Deck Profile

`product-platform`

Reason: the deck is a product-platform narrative for a reusable company-level platform, not a single product launch or a pure engineering architecture review.

## Secondary Gates

- `strategy-leadership`: management-level立项会，需要清晰的战略必要性、投资价值和结论收束。
- `engineering-platform`: diagrams must keep Agent, experiment, data, model, tool, permission, log, and audit semantics accurate enough for technical review.

## Required Proof Objects

- Platform map: industry product lines on top, four common modules in the middle, model/data/experiment/tool/enterprise systems beneath.
- Use-case/workflow visual: Agent demo flow from task intake to planning, tool/knowledge/model calls, result, logging, audit.
- Adoption/expansion proof: quantified speed and reuse metrics.
- Product-to-financial linkage: repeated engineering cost reduction, rollout cycle compression, toB delivery scale.
- Strategic direction: product flywheel and final立项判断.

## Source And Asset Requirements

- Use only the provided Markdown as content source for v1.
- Do not use previous product files except as broad workspace background.
- Do not invent customer names, reported financial numbers, or official brand assets.

## QA Gates

- 12-14 slides for a 30-minute report.
- Every slide needs a claim and proof object, not only narrative text.
- Product-system diagrams must have explicit lanes or arrows.
- Avoid generic feature-card grids; overview pages must encode platform relationships or business outcomes.
- Text must remain readable in a 16:9 browser presentation and when printed to PDF.
