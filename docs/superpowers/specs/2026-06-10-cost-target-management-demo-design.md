# 成本管理与目标管理复杂 DEMO 设计

## 目标

在产品管理平台中补齐复杂版 `成本管理` 和新增 `目标管理` 顶部 Tab。这个 DEMO 不追求完整生产系统，但必须让观看者相信系统已经具备经营管理复杂度：可以从产品、商品、计费项一路追踪到成本模型、分摊规则、季度目标、目标达成、成本使用和毛利风险。

最终顶部导航顺序：

- 产品管理
- 商品管理
- 成本管理
- 目标管理
- 费用中心

## 参考原则

SPEC 采用以下最佳实践作为设计约束：

- FinOps unit economics 强调把单位收入、单位成本和毛利连接起来，用于判断产品、客户和资源使用是否创造利润。参考：FinOps Foundation Cloud Unit Economics（https://www.finops.org/wg/introduction-cloud-unit-economics/）。
- 成本分摊需要通过层级、标签、业务 owner、成本中心、产品或应用等维度把成本归属到责任对象，并处理共享成本。参考：FinOps Foundation Cloud Cost Allocation Guide（https://www.finops.org/wg/cloud-cost-allocation/）。
- 云成本管理需要建立组织指标、成本归因分类，并按照 workload metrics 分摊成本。参考：AWS Well-Architected COST 3（https://docs.aws.amazon.com/wellarchitected/latest/framework/cost-03.html）。
- 成本管理不是纯技术账单，应该对齐组织财务目标、预算和预测。参考：AWS Cloud Financial Management（https://docs.aws.amazon.com/wellarchitected/latest/framework/cost-cfm.html）。
- 目标管理采用预算/目标 vs 实际的 FP&A 口径，按月度或季度解释差异、判断 favorable / negative variance。参考：Wall Street Prep Budget to Actual Variance Analysis（https://www.wallstreetprep.com/knowledge/budget-actual-variance-analysis-fpa/）。
- SaaS 毛利率依赖一致的 COGS 口径，不一致的成本定义会导致毛利率无法解释。参考：SaaS Capital COGS discussion（https://www.saas-capital.com/blog-posts/what-should-be-included-in-cogs-for-my-saas-business/）。

## 边界

`费用中心` 表达客户实例和账单记录，偏“发生了什么收费事件”。

`成本管理` 表达成本项、成本模型、分摊规则和版本，偏“这些商品/计费项应该按什么成本口径计算”。

`目标管理` 表达产品季度经营目标、目标达成、成本预算使用和毛利风险，偏“经营目标是否达成，以及为什么”。

`商品详情 > 成本与折扣` 继续保留为单个商品的只读结果表，由当前生效成本模型派生，不承担成本模型治理。

目标管理必须作为产品平台顶层 Tab，而不是成本管理的左侧菜单。原因是目标管理同时消费账单、成本模型、产品、商品和预测信息，职责高于单一成本治理页面；把它放在成本管理内会让“目标定义”和“成本口径定义”边界混淆。

`成本管理` 中出现的实际收入、实际成本和实际毛利率只用于总览/分析视图，来源分别是费用中心账单汇总、已确认成本和分摊结果。`成本模型` 本身只维护目标成本、预测成本、成本拆解和试算毛利，不作为账单事实来源。

## 计算口径

数据快照日：

`2026-06-10`。

时间状态：

- 2026 Q2 是进行中目标，可以展示已达成业绩、已确认成本、实际毛利额、实际毛利率和预测达成率。
- 2026 Q3 是未开始/规划目标，actual 字段显示 `0` 或 `-`，主要展示业绩目标、成本预算、目标毛利率和预测达成率，不展示看似已经发生的收入或成本。

所有金额使用 CNY，展示为 `¥1,234,567`。

业绩目标口径：

`账单金额口径`。DEMO 中目标管理的业绩目标和达成值都按费用中心账单可解释的金额口径展示，不使用签约额或确认收入口径，避免引入合同、收入确认和回款系统。

成本口径：

`目标成本` 来自成本模型和分摊规则；`已确认成本` 来自账单周期内的资源、人力、交付和共享成本 mock 汇总。私有化部署没有实例，但仍可通过人工导入账单和项目成本进入目标管理。

COGS 口径：

- SaaS：计入资源、存储、模型推理、平台运维摊销、L1/L2 支持、CSM 和第三方服务成本；不计入销售、市场、研发投入和管理费用。
- 私有化部署：计入交付实施、人力服务、专属环境资源、年度维保、第三方服务和项目共享成本；不计入销售、市场和通用管理费用。
- 人力服务：计入 L1、L2、CSM、交付人天和项目管理摊销。
- 维保：计入 L1/L2 支持、SLA 响应、版本升级交付和运维摊销。

目标达成率：

`已达成业绩 / 业绩目标`

成本使用率：

`已确认成本 / 成本预算`

实际毛利额：

`已达成业绩 - 已确认成本`

实际毛利率：

`实际毛利额 / 已达成业绩`

除零规则：`已达成业绩 = 0` 时，实际毛利率显示 `-`，不参与毛利率风险判断。

目标毛利额：

`业绩目标 - 成本预算`

目标毛利率：

`目标毛利额 / 业绩目标`

预测达成率：

`预测季度业绩 / 业绩目标`

毛利率差异：

`实际毛利率 - 目标毛利率`

风险状态建议规则：

- 正常：达成率 >= 85%，且实际毛利率 >= 目标毛利率 - 5pp。
- 关注：65% <= 达成率 < 85%，或成本使用率 > 达成率 + 10pp。
- 风险：达成率 < 65%，或实际毛利率 < 目标毛利率 - 10pp，或成本使用率 > 100%。

未开始季度只按预测达成率和目标状态展示风险，不使用 actual 毛利率判断。

成本相关记录至少包含：

- `productId`
- `commodityId`
- `billingItemCode`
- `costModelId`
- `costVersion`
- `allocationRuleId`，仅分摊相关记录需要

目标相关记录至少包含：

- `targetId`
- `productId`
- `commodityId`，商品贡献和目标详情商品贡献需要
- `billingItemCode`，商品贡献的主要计费项需要
- `costModelId`
- `costVersion`

实现时允许 UI 只展示名称和编号，但数据结构必须保留追踪键。虚拟细胞 SaaS 的 7 个计费项必须复用现有计费项编号，不重新编造另一套编号。

## 交互要求

- 顶部 Tab 切换必须保持产品平台全屏布局。
- 成本管理和目标管理都必须有左侧导航。
- 大表格横向空间不足时使用横向滚动，不压缩到不可读。
- 列表页要有筛选器、搜索框、主操作按钮和表格。
- 复杂列表要有分页，每页 10 条。
- 所有新增、查看、编辑类按钮可以显示暂未接入提示；详情页中的“查看”应尽量实现为只读详情，而不是全部提示。
- 不允许在展示类页面提供新增、编辑、删除真实功能。
- UI 风格要延续现有产品平台：低饱和、操作系统感、表格密度高、卡片半径不超过现有风格。
